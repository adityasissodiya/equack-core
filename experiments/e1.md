Experiment E1: Convergence Under Random Ordering - Design

  Overview

  This experiment validates Theorem 1 (operation delivery order does not affect final state) by:
  1. Generating a canonical 100K-op log using workload W2 (concurrent writers, 10 authors)
  2. Creating 100 causally-valid permutations of the operation sequence
  3. Replaying each permutation on a fresh DAG
  4. Comparing BLAKE3 state digests across all replicas

  Key Design Decisions

  1. Causally-Valid Permutations

  The existing test in replay_prop.rs:64-68 uses arbitrary random shuffles of ops, which works because the DAG's insert() method handles pending staging automatically (ops
  wait for missing parents). However, for the experiment's description ("respecting causal dependencies"), we have two options:

  - Option A (Simple): Use arbitrary permutations and rely on the DAG's pending staging. This is what the existing tests do.
  - Option B (Explicit): Generate permutations where parents always precede children (topological orderings).

  I recommend Option A for two reasons:
  1. It tests the DAG's pending staging mechanism, which is part of the system's correctness
  2. The experiment's invariant (identical digests) holds regardless of insertion order due to deterministic topo-sort

  2. Workload W2: Concurrent Writers

  The existing gen_concurrent_writers function in bench.rs:264-300 generates operations from N authors (clamped to [2,8]). For 10 concurrent writers, we need to extend this
  to support peers=10.

  3. Implementation Location

  I recommend adding this as:
  - A new integration test in crates/core/tests/convergence_e1.rs for automated CI verification
  - A new CLI scenario convergence in bench.rs for benchmarking and artifact generation

  Implementation Plan

  File: crates/core/tests/convergence_e1.rs

  //! E1: Convergence Under Random Ordering
  //! Validates that operation delivery order does not affect final state.

  use ecac_core::crypto::{generate_keypair, vk_to_bytes};
  use ecac_core::dag::Dag;
  use ecac_core::hlc::Hlc;
  use ecac_core::op::{Op, OpId, Payload};
  use ecac_core::replay::replay_full;
  use rand::{rngs::StdRng, seq::SliceRandom, SeedableRng};
  use std::collections::HashSet;

  const N_OPS: usize = 100_000;
  const N_AUTHORS: usize = 10;
  const N_PERMUTATIONS: usize = 100;
  const MASTER_SEED: u64 = 0xE1_CONVERGENCE;

  /// Generate deterministic keypair from seed and label
  fn key_pair(seed: u64, label: &[u8]) -> ed25519_dalek::SigningKey {
      let mut bytes = [0u8; 32];
      let mut input = [0u8; 16];
      input[..8].copy_from_slice(&seed.to_le_bytes());
      let h = blake3::hash(&[&input, label].concat());
      bytes.copy_from_slice(&h.as_bytes()[..32]);
      ed25519_dalek::SigningKey::from_bytes(&bytes)
  }

  /// Generate W2 workload: N_AUTHORS concurrent writers, each with independent parent chains
  fn gen_w2_workload(seed: u64, n_ops: usize, n_authors: usize) -> Vec<Op> {
      let sks: Vec<_> = (0..n_authors)
          .map(|i| key_pair(seed, format!("w2/{i}").as_bytes()))
          .collect();
      let pks: Vec<[u8; 32]> = sks
          .iter()
          .map(|sk| vk_to_bytes(&sk.verifying_key()))
          .collect();

      let mut parents: Vec<Vec<OpId>> = vec![Vec::new(); n_authors];
      let mut ops = Vec::with_capacity(n_ops);

      for i in 0..n_ops {
          let author = i % n_authors;
          let sk = &sks[author];
          let pk = pks[author];

          // Mix of MVReg and ORSet operations
          let (key, value) = if i % 3 == 0 {
              (format!("set+:o:items:item{}", i % 100), vec![(i % 256) as u8])
          } else if i % 3 == 1 {
              (format!("set-:o:items:item{}", (i + 50) % 100), vec![])
          } else {
              ("mv:o:counter".to_string(), format!("v{i}").into_bytes())
          };

          let payload = Payload::Data { key, value };
          let hlc = Hlc::new(1_000 + i as u64, (author as u32) + 1);

          let op = Op::new(parents[author].clone(), hlc, pk, payload, sk);
          parents[author] = vec![op.op_id];
          ops.push(op);
      }

      ops
  }

  /// Replay ops in given order and return state digest
  fn replay_in_order(ops: &[Op], order: &[usize]) -> [u8; 32] {
      let mut dag = Dag::new();
      for &idx in order {
          dag.insert(ops[idx].clone());
      }
      let (_state, digest) = replay_full(&dag);
      digest
  }

  #[test]
  #[ignore] // Run with: cargo test -p ecac-core --test convergence_e1 -- --ignored --nocapture
  fn e1_convergence_under_random_ordering() {
      println!("E1: Generating canonical log ({} ops, {} authors)...", N_OPS, N_AUTHORS);
      let ops = gen_w2_workload(MASTER_SEED, N_OPS, N_AUTHORS);

      // Reference digest: sequential insertion order
      println!("E1: Computing reference digest (sequential order)...");
      let reference_order: Vec<usize> = (0..ops.len()).collect();
      let reference_digest = replay_in_order(&ops, &reference_order);
      println!("E1: Reference digest: {}", hex::encode(reference_digest));

      // Generate and test N_PERMUTATIONS random orderings
      let mut all_match = true;
      let mut digests: HashSet<[u8; 32]> = HashSet::new();
      digests.insert(reference_digest);

      for perm_idx in 0..N_PERMUTATIONS {
          let perm_seed = MASTER_SEED.wrapping_add(perm_idx as u64);
          let mut rng = StdRng::seed_from_u64(perm_seed);

          let mut order: Vec<usize> = (0..ops.len()).collect();
          order.shuffle(&mut rng);

          let digest = replay_in_order(&ops, &order);
          digests.insert(digest);

          if digest != reference_digest {
              eprintln!("E1 FAIL: Permutation {} produced different digest!", perm_idx);
              eprintln!("  Expected: {}", hex::encode(reference_digest));
              eprintln!("  Got:      {}", hex::encode(digest));
              all_match = false;
          }

          if (perm_idx + 1) % 10 == 0 {
              println!("E1: Verified {}/{} permutations", perm_idx + 1, N_PERMUTATIONS);
          }
      }

      println!("\nE1 Results:");
      println!("  Total permutations: {}", N_PERMUTATIONS);
      println!("  Unique digests: {}", digests.len());
      println!("  All match reference: {}", all_match);

      assert!(all_match, "E1 FAILED: Some permutations produced different digests");
      assert_eq!(digests.len(), 1, "E1 FAILED: Expected exactly 1 unique digest");

      println!("E1 PASSED: All {} permutations converged to same state", N_PERMUTATIONS);
  }

  Expected Results

  Given the system's design:
  1. All 100 digests should be identical because:
    - DAG's topo_sort() uses deterministic (HLC, OpId) ordering
    - CRDT operations (MVReg, ORSet) are commutative
    - State digest uses deterministic JSON serialization
  2. Performance: Each replay should complete in similar time (~proportional to N_OPS)

  Failure Modes to Test

  The experiment implicitly tests:
  1. Topo-sort determinism: Different insertion orders produce same topo order
  2. Pending staging correctness: Out-of-order insertions resolve correctly
  3. CRDT commutativity: MVReg and ORSet merge correctly regardless of apply order
  4. Digest determinism: Same state always produces same BLAKE3 hash
